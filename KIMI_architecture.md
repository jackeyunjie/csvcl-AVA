## 一、架构设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **渐进式演进** | 基于现有MT5_AI_Trading代码库逐步增强，避免推倒重来 |
| **模块化设计** | 各功能模块独立部署、独立升级，通过标准接口通信 |
| **事件驱动** | 核心数据流采用事件驱动架构，支持异步处理和实时响应 |
| **多Agent协作** | 参考TradingAgents和Vibe-Trading，采用LangGraph角色化Agent设计 |
| **MCP生态** | 遵循Model Context Protocol标准，与Claude/Cursor等AI助手无缝集成 |
| **数据一致性** | 关键状态数据统一存储于DuckDB，确保多Agent共享一致视图 |

### 1.2 技术选型决策

| 决策项 | 选择方案 | 核心理由 |
|--------|----------|----------|
| MT5对接 | **ZeroMQ分布式架构** | 已有成熟实现，PUB/SUB+REQ/REP双通道，微秒级延迟 |
| AI技术栈 | **混合方案** | 核心决策自研 + LLM分析接入OpenAI/Claude/DeepSeek API |
| 回测引擎 | **VectorBT向量化引擎** | GitHub星标最高，Numba/Rust加速，10,000参数组合秒级完成 |
| Agent框架 | **LangGraph** | TradingAgents采用，支持状态机、循环、条件分支，比CrewAI更灵活 |
| 数据存储 | **DuckDB + Parquet** | 已有DuckDB基础，Parquet用于历史数据归档 |
| 消息总线 | **Python asyncio Queue + Redis** | 单机Queue满足当前需求，Redis用于扩展 |
| Web前端 | **React 19 + TypeScript + Vite** | 现代化前端栈，支持实时数据流和复杂可视化 |
| API框架 | **FastAPI** | 高性能异步API，自动生成OpenAPI文档 |
| MCP协议 | **MCP标准** | Vibe-Trading已验证，36工具暴露，Claude/Cursor原生支持 |

### 3.5 核心引擎层

#### 3.5.1 State Hex引擎（已有增强）
- **现有能力**: D1/W1/MN1三元组 + H4/H1五元组状态计算

  - 订阅机制：Agent可订阅特定品种的State Hex变化

  - 异步IO避免阻塞主线程

#### 3.5.2 收缩检测引擎（已有增强）
- **现有能力**: ACD枢轴收缩检测
- **增强内容**:

#### 3.5.3 回测引擎（VectorBT集成）
- **架构**: 向量化回测（Vectorized Backtesting）
- **核心设计**:
  ```python
  import vectorbt as vbt

  class VectorBTBacktestEngine:
      def __init__(self, data: pd.DataFrame, strategy: Strategy):
          self.data = data  # 多周期对齐数据
          self.strategy = strategy
      
      def run(self) -> BacktestResult:
          # 1. 预计算所有指标（向量化）
          indicators = self._calculate_indicators()
          # 2. 生成信号序列（向量化）
          entries, exits = self._generate_signals(indicators)
          # 3. 使用VectorBT模拟持仓和盈亏
          pf = vbt.Portfolio.from_signals(
              self.data['close'], entries, exits,
              init_cash=self.config.initial_balance,
              fees=self.config.commission
          )
          # 4. 计算绩效指标
          metrics = pf.stats()
          return BacktestResult(metrics, pf)
  ```
- **性能目标**: H1数据5年量< 10秒

| 存储类型 | 技术 | 用途 | 数据模型 |

| 状态数据库 | DuckDB | 实时状态、观察记录、Agent状态 | 关系型 |
| 历史数据 | Parquet | OHLCV历史数据、回测结果 | 列式存储 |
| 缓存 | SQLite/Redis | 计算缓存、会话数据 | KV存储 |
| 日志 | 文件/Elasticsearch | 系统日志、交易日志、审计日志 | 时序 |

#### 3.6.2 消息总线
- **单机模式**: Python asyncio Queue
- **扩展模式**: Redis Pub/Sub
- **事件类型**:
  - `TICK_RECEIVED` - 收到新Tick
  - `STATE_UPDATED` - State Hex更新
  - `CONTRACTION_DETECTED` - 检测到收缩
  - `BREAKOUT_CONFIRMED` - 突破确认
  - `SIGNAL_GENERATED` - 信号生成
  - `ORDER_EXECUTED` - 订单执行
  - `RISK_ALERT` - 风险告警

#### 3.6.3 MT5桥接层
- **复用现有**: `mt5_bridge.py`, `mt5_bridge_dual.py`
- **增强内容**:
  - 断线重连：指数退避重试，最大重试10次
  - 心跳检测：每5秒PING/PONG，超时500ms判定断线
  - 连接池：支持多个MT5终端连接
  - 数据缓冲：断线期间缓存数据，恢复后补发
