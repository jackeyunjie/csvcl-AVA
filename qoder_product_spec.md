## 一、文档概述

### 1.1 目的
本文档详细描述Qoder AI量化多周期视角交易平台的产品功能规格，涵盖自然语言策略生成、智能回测、多Agent协作、实时市场分析四大核心功能模块的具体实现要求和用户交互方式。

### 1.2 适用范围
本文档面向产品经理、系统架构师、开发工程师、测试工程师及项目管理人员，作为系统设计、开发、测试和验收的基准依据。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| State Hex | 多周期状态编码系统，基于压缩/趋势/位置/波动率四个维度生成16进制状态码 |
| 收缩(Squeeze) | 价格波动区间收窄、波动率下降的市场状态 |
| 突破(Breakout) | 价格突破收缩区间边界，伴随成交量放大 |
| 趋势跟踪(Trend Following) | 突破后沿趋势方向持仓的交易策略 |
| 多周期共振 | 多个时间周期同时发出同向信号的状态 |
| Agent | 具备独立决策能力的AI智能体，负责特定任务域 |

## 二、产品定位与愿景

### 3.2 自然语言策略生成模块 (NL-STRATEGY)

#### 3.2.1 功能概述
用户通过自然语言对话描述交易策略意图，系统自动解析策略逻辑、生成可执行代码，并提供策略语法验证和逻辑校验。

#### 3.2.2 功能需求

**FR-NL-001 策略意图解析**
- 输入：自然语言描述（如"创建一个均线金叉策略，MA5上穿MA20时买入，跌破MA10时卖出"）
- 处理：
  1. 使用LLM提取策略要素（指标、条件、动作、参数）
  2. 映射到预定义策略模板库
  3. 识别策略类型（趋势跟踪/均值回归/突破/套利）
- 输出：结构化策略描述JSON

**FR-NL-002 策略代码生成**
- 支持生成目标格式：
  - Python策略代码（用于回测引擎）
  - MQL5 EA代码（用于MT5实盘）
  - Pine Script v6（用于TradingView）
  - TDX公式（用于通达信/同花顺）
- 代码生成要求：
  - 包含完整的初始化、指标计算、信号生成、订单管理逻辑
  - 自动注入风控参数（止损、止盈、仓位管理）
  - 代码注释清晰，变量命名规范

**FR-NL-003 策略验证**
- 语法验证：检查生成代码的语法正确性
- 逻辑校验：
  - 检测死循环和无限递归
  - 验证指标参数合理性（如MA周期不能为负）
  - 检查条件互斥性（买入和卖出条件不能同时触发）
- 回测预检：快速运行1个月数据验证策略可执行性

**FR-NL-004 策略模板库**
- 内置模板分类：
  - 趋势类：均线交叉、MACD、ADX趋势
  - 反转类：RSI超买超卖、布林带反转、枢轴点反转
  - 突破类：通道突破、波动率突破、成交量突破
  - 多因子类：State Hex共振、收缩突破、资金流确认
- 支持用户保存自定义模板

**FR-NL-005 策略参数优化建议**
- 基于历史数据自动推荐参数范围
- 提供参数敏感性分析
- 生成参数热力图（如MA5-MA20组合的夏普比率分布）

#### 3.2.3 用户交互流程

```
用户输入: "创建一个基于State Hex多周期共振的策略，
          当D1/W1/MN1同时出现 bullish_trend 时买入，
          任一周期转为 bearish_trend 时卖出"

系统处理:
  ↓ 意图解析 → 识别为"多周期共振趋势跟踪策略"
  ↓ 模板匹配 → 匹配 "state_hex_resonance_trend" 模板
  ↓ 参数提取 → timeframe=[D1,W1,MN1], condition=[bullish_trend], action=[BUY/SELL]
  ↓ 代码生成 → 生成Python/MQL5/Pine Script三种格式
  ↓ 策略验证 → 语法通过，逻辑校验通过，预检通过
  ↓ 结果呈现 → 展示策略代码 + 可视化流程图 + 参数面板

用户操作:
  → 调整参数 → 实时更新代码
  → 保存模板 → 存入个人模板库
  → 一键回测 → 跳转回测模块
```

#### 3.2.4 界面原型要点

| 界面元素 | 说明 |
|----------|------|
| 对话输入框 | 支持自然语言输入，提供策略示例快捷按钮 |
| 策略卡片 | 展示解析后的策略要素（指标、条件、动作） |
| 代码编辑器 | 语法高亮，支持Python/MQL5/Pine Script切换 |
| 参数面板 | 滑块/数字输入调整参数，实时预览影响 |
| 验证状态 | 绿/黄/红三色标识验证结果 |

### 3.3 智能策略回测模块 (BACKTEST)

#### 3.3.1 功能概述
提供高性能、多维度的策略回测能力，支持30秒内返回完整回测结果，包含详细的风险分析报告和绩效指标。

#### 3.3.2 功能需求

**FR-BT-001 一键回测**
- 用户操作：点击"回测"按钮或调用API
- 系统响应：
  1. 接收策略代码 + 回测参数
  2. 加载历史数据（自动匹配品种和周期）
  3. 执行回测引擎
  4. 生成报告（目标：30秒内完成）
- 进度反馈：实时显示当前阶段（数据加载→回测执行→报告生成）

**FR-BT-002 多周期数据对齐**
- 支持周期：Tick, M1, M5, M15, M30, H1, H4, D1, W1, MN1
- 对齐方式：
  - 时间戳对齐：以最小周期为基准，大周期数据前向填充
  - State Hex对齐：各周期独立计算state_hex后按时间戳合并
  - 事件对齐：以特定事件（如突破点）为锚点对齐各周期数据
- 数据质量检查：缺失值处理、异常值检测、数据连续性验证

**FR-BT-003 真实交易环境模拟**
- 滑点模拟：
  - 固定滑点模式（用户指定点数）
  - 波动率自适应滑点（基于ATR动态计算）
  - 流动性分层滑点（根据订单量和市场深度）
- 手续费计算：
  - 按品种配置点差、佣金、隔夜利息
  - 支持按交易量阶梯计费
- 成交模拟：
  - 即时成交（理想情况）
  - 限价单成交（价格触达才成交）
  - 部分成交（大单拆分为多笔）

**FR-BT-004 绩效指标计算**

| 指标类别 | 具体指标 |
|----------|----------|
| 收益类 | 总收益率、年化收益率、超额收益、信息比率 |
| 风险类 | 最大回撤、波动率、VaR(95%)、CVaR、下行偏差 |
| 效率类 | 夏普比率、索提诺比率、卡玛比率、欧米伽比率 |
| 交易类 | 胜率、盈亏比、平均盈利/亏损、最大连续盈亏次数 |
| 持仓类 | 平均持仓时间、持仓周转率、资金利用率 |

**FR-BT-005 风险分析报告**
- 报告内容：
  - 收益曲线与回撤曲线
  - 月度/年度收益热力图
  - 交易分布统计（入场时间、持仓时长、盈亏分布）
  - 风险指标雷达图
  - 蒙特卡洛模拟结果（1000次随机打乱交易顺序）
  - Walk-Forward分析（滚动窗口内外样本验证）
- 输出格式：HTML交互报告 + PDF导出 + JSON数据

**FR-BT-006 多策略对比**
- 支持同时回测最多5个策略
- 对比维度：收益曲线叠加、绩效指标表格、风险散点图
- 生成策略排名和推荐结论

**FR-BT-007 参数优化**
- 优化算法：网格搜索、随机搜索、贝叶斯优化（Optuna）
- 优化目标：夏普比率最大化、回撤最小化、卡玛比率最大化
- 约束条件：最大回撤<20%、最少交易次数>30次
- 输出：最优参数组合 + 参数敏感性分析图

#### 3.3.3 回测引擎性能要求

| 指标 | 要求 |
|------|------|
| 回测速度 | H1数据5年量（约43800根K线）< 10秒 |
| 并发回测 | 支持5个策略同时回测 |
| 内存占用 | 单次回测< 2GB |
| 数据缓存 | 历史数据本地缓存，重复回测秒级响应 |

### 3.4 多Agent协作系统模块 (AGENT)

#### 3.4.1 功能概述
构建模拟真实交易公司组织架构的多Agent协作系统，各Agent具备独立决策能力，通过结构化协议进行协作。

#### 3.4.2 Agent角色定义

**AG-001 研究Agent (Research Agent)**
- 职责：市场研究、策略发现、信号生成
- 子类型：
  - 技术分析师：基于技术指标和Price Action分析
  - 基本面分析师：分析财报、宏观数据、行业趋势
  - 情绪分析师：分析新闻、社交媒体情绪
  - 量化研究员：因子挖掘、统计套利、机器学习模型
- 输入：市场数据、新闻流、历史回测结果
- 输出：研究报告、交易信号、策略建议
- 协作方式：向交易员Agent提交交易建议，向风控Agent报告风险因素

**AG-002 执行Agent (Execution Agent)**
- 职责：订单执行、滑点控制、成交优化
- 功能：
  - 订单拆分（大单拆分为多笔小单）
  - 智能下单时机选择（基于市场微观结构）
  - 紧急平仓执行
- 输入：交易员Agent的订单指令
- 输出：成交确认、执行报告、滑点分析
- 协作方式：接收交易员Agent指令，向风控Agent报告大额成交

**AG-003 风控Agent (Risk Agent)**
- 职责：实时监控风险、触发风控措施、生成风险报告
- 监控维度：
  - 账户级：净值回撤、保证金使用率、日亏损限额
  - 品种级：单一品种敞口、相关性风险
  - 策略级：策略衰减检测、信号质量监控
  - 系统级：连接状态、数据延迟、异常交易检测
- 风控措施：
  - 预警：邮件/短信通知
  - 限制：暂停新开仓
  - 强制：平仓减仓
  - 紧急：全平+系统锁定
- 输入：所有Agent的状态报告、市场数据、账户信息
- 输出：风险评分、风控指令、风险报告

**AG-004 交易员Agent (Trader Agent)**
- 职责：综合各方信息做出交易决策
- 决策流程：
  1. 收集研究Agent的信号和建议
  2. 评估信号质量（参考SignalScorer历史表现）
  3. 查询风控Agent的风险限额
  4. 生成交易指令（品种/方向/数量/价格类型）
  5. 提交执行Agent执行
- 输入：研究Agent报告、风控Agent限额、账户状态
- 输出：交易指令、交易日志、决策理由

**AG-005 组合经理Agent (Portfolio Agent)**
- 职责：组合层面资产配置、策略权重调整、绩效归因
- 功能：
  - 基于Kelly准则和风险预算的仓位分配
  - 策略相关性分析和分散化建议
  - 组合再平衡触发和执行
- 输入：各策略绩效、市场环境、风险预算
- 输出：组合配置方案、再平衡指令、绩效归因报告

**AG-006 观察Agent (Observer Agent)** - 特色Agent
- 职责：专门追踪收缩-突破-趋势全生命周期
- 功能：
  - 收缩识别：监控多周期收缩状态（ACD枢轴、BB Width、Kaufman Width）
  - 突破确认：检测突破信号（价格突破、成交量确认、State Hex变化）
  - 趋势跟踪：突破后的趋势持续监控和持仓管理
  - 生命周期记录：完整记录每个收缩→突破→趋势案例
- 输出：观察报告、突破预警、趋势状态更新

#### 3.4.3 Agent协作协议

```
协作消息格式:
{
  "msg_id": "uuid",
  "from_agent": "research_technical",
  "to_agent": "trader",
  "msg_type": "SIGNAL",
  "priority": 1,  // 1=紧急, 2=高, 3=中, 4=低
  "timestamp": "2026-06-06T10:30:00Z",
  "payload": {
    "signal_type": "BUY",
    "symbol": "EURUSD",
    "confidence": 0.85,
    "reasoning": "D1 State Hex = A (bullish_trend + expansion), W1 State Hex = 2 (contraction_breakout), 多周期共振",
    "suggested_entry": 1.0850,
    "suggested_sl": 1.0800,
    "suggested_tp": 1.0950,
    "timeframe": "H1",
    "expiration": "2026-06-07T10:30:00Z"
  },
  "requires_ack": true
}
```

#### 3.4.4 协作流程示例

```
[收缩识别阶段]
Observer Agent → 检测到EURUSD在D1/W1同时收缩
  ↓ 发送 CONTRACTION_ALERT 给 Research Agent
Research Agent → 分析收缩特征，确认高概率突破方向
  ↓ 发送 RESEARCH_REPORT 给 Trader Agent

[突破确认阶段]
Observer Agent → 检测到价格突破收缩区间上沿 + 成交量放大3倍
  ↓ 发送 BREAKOUT_CONFIRMED 给 Trader Agent
Trader Agent → 综合Research报告 + 突破确认，生成买入信号
  ↓ 发送 RISK_CHECK 给 Risk Agent
Risk Agent → 检查账户风险限额 → 通过
  ↓ 发送 APPROVED 给 Trader Agent
Trader Agent → 生成订单指令
  ↓ 发送 EXECUTE_ORDER 给 Execution Agent
Execution Agent → 执行订单 → 返回成交确认

[趋势跟踪阶段]
Observer Agent → 持续监控趋势状态
  ↓ 当检测到趋势减弱信号时发送 TREND_WEAKENING
Trader Agent → 评估是否减仓/平仓
  ↓ 发送调整指令给 Execution Agent
```

### 3.5 实时市场分析模块 (ANALYSIS)

#### 3.5.1 功能概述
提供实时多维度市场数据分析，重点关注多周期结构共振、收缩突破追踪、行业轮动和品种状态评估。

#### 3.5.2 功能需求

**FR-AN-001 多周期结构共振分析**
- 共振检测：
  - 扫描所有监控品种的MN1/W1/D1/H4/H1五元组State Hex
  - 识别同向状态共振（如D1/W1/MN1同时为bullish_trend）
  - 计算共振强度评分（基于周期权重和状态一致性）
- 共振类型：
  - 趋势共振：多周期同向趋势
  - 收缩共振：多周期同时收缩（高概率突破前兆）
  - 突破共振：多周期同时突破
  - 反转共振：大周期趋势+小周期反转信号
- 输出：共振热力图、共振品种排名、共振强度时间序列

**FR-AN-002 收缩突破追踪**
- 收缩监控面板：
  - 实时显示各品种各周期的收缩状态
  - 收缩持续时间计数器
  - 收缩强度评分（基于BB Width、Kaufman Width、ATR）
- 突破预警：
  - 当收缩持续N根K线后发出预警
  - 突破方向概率预测（基于历史统计）
  - 突破目标位估算（基于收缩区间高度投影）
- 生命周期记录：
  - 每个收缩→突破→趋势案例的完整记录
  - 记录触发条件、突破时间、趋势持续时间、最终收益
  - 支持案例检索和模式匹配（"找出与当前EURUSD状态相似的历史案例"）

**FR-AN-003 行业轮动跟踪**
- 板块分类：外汇、股指、商品、加密货币
- 轮动指标：
  - 各板块State Hex分布统计
  - 板块强弱排名（基于平均收益率、趋势强度）
  - 资金流向分析（基于moneyflow_energy_layer）
- 轮动信号：
  - 板块强弱转换预警
  - 资金流入/流出板块识别
  - 推荐配置板块权重

**FR-AN-004 品种状态评估**
- 状态卡片：每个品种一个综合状态卡片
- 卡片内容：
  - 当前价格、涨跌幅、波动率
  - 五元组State Hex可视化
  - 收缩/突破/趋势状态标识
  - 资金流能量标签
  - 最近信号历史
  - 绩效统计（该品种近期信号胜率）
- 状态变化提醒：当品种状态发生重要变化时推送通知

**FR-AN-005 实时数据流**
- 数据源：MT5 ZeroMQ实时行情
- 更新频率：Tick级接收，H1/M15级别分析
- 数据缓冲：最近1000根K线内存缓存
- 异常检测：价格跳空、数据缺失、连接中断自动恢复

## 四、非功能需求

### 4.1 性能需求

| 指标 | 目标值 |
|------|--------|
| 回测响应时间 | 单次回测< 30秒 |
| 实时数据延迟 | 从MT5到系统< 100ms |
| 并发用户 | 支持10个用户同时操作 |
| 系统启动时间 | 从启动到可交易< 60秒 |
| 数据查询响应 | 1年数据查询< 2秒 |

### 4.2 可靠性需求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 99.5%（允许每周< 1小时维护） |
| 数据完整性 | 交易数据零丢失 |
| 故障恢复 | MT5断线后< 30秒自动重连 |
| 回测一致性 | 同一策略同一数据多次回测结果一致 |

### 4.3 安全需求

- API密钥加密存储（AES-256）
- 交易指令签名验证
- 敏感操作双因素认证
- 审计日志完整记录

### 4.4 可扩展性需求

- 支持新增品种无需修改代码（配置化）
- 支持新增策略类型（插件化）
- 支持新增Agent角色（模块化）
- 支持新增数据源（适配器模式）
