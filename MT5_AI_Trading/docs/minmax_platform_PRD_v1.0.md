> 版本: v1.0 (minmax标识)

## 1. 产品愿景与目标

### 1.1 核心愿景

> **"让每位交易者都能拥有AI量化研究员的能力，通过多周期共振视角洞察市场"**

MiniMax平台是一个具有高度科技感的AI量化交易系统，融合了：
- **自然语言策略生成** — 用对话方式快速创建量化策略
- **智能快速回测** — 30秒内获得完整绩效报告
- **多Agent协作决策** — 研究/执行/风控三权分立
- **多周期共振分析** — State Hex + 收缩突破 + 趋势跟踪

### 1.2 与现有系统的协同

本平台基于以下现有资产构建，实现1+1>2的效果：

| 现有资产 | 复用方式 | 新增价值 |
|----------|----------|----------|
| MT5_AI_Trading回测平台 | 数据层/计算层/执行层 | 快速回测能力 |
| State Hex Engine | 核心状态编码引擎 | 多周期共振判断 |
| MT5Bridge (ZeroMQ) | 实时行情/交易通道 | 实盘部署能力 |
| process_real_mt4_data.py | 数据处理流程 | 真实数据接入 |
| csv_color_marker.py | 可视化标记 | 科技感UI元素 |
| EmailSender | 通知系统 | 多渠道告警 |

## 3. 功能规格说明

### 3.1 自然语言策略生成模块

#### 3.1.1 功能描述

用户通过对话方式描述交易想法，系统自动转换为可执行的策略代码。

#### 3.1.2 用户交互方式

**对话式输入示例**:
```
用户: "创建一个均线金叉策略，MA5上穿MA20时买入，跌破MA10时卖出"

系统: ┌─────────────────────────────────────────────────────────┐
      │ 🌐 策略生成中...                                       │
      │                                                         │
      │ 策略名称: 均线金叉策略_v1                               │
      │ 方向: 做多                                             │
      │ 周期: H1                                               │
      │                                                         │
      │ 代码预览:                                               │
      │ ┌─────────────────────────────────────────────────┐    │
      │ │ class MaCrossStrategy(Strategy):                │    │
      │ │     def __init__(self, fast=5, slow=20):        │    │
      │ │         self.fast = fast                        │    │
      │ │         self.slow = slow                        │    │
      │ │                                                 │    │
      │ │     def on_bar(self, bar):                       │    │
      │ │         if ma5 > ma20 and prev_ma5 <= prev_ma20: │    │
      │ │             self.buy(bar.close)                  │    │
      │ │         elif ma10 < bar.close:                  │    │
      │ │             self.sell()                         │    │
      │ └─────────────────────────────────────────────────┘    │
      │                                                         │
      │ [立即回测] [修改参数] [保存草稿] [生成可视化]            │
      └─────────────────────────────────────────────────────────┘
```

**支持的策略模板**:
| 模板ID | 名称 | 参数 | 示例 |
|--------|------|------|------|
| `ma_crossover` | 均线交叉 | fast_ma, slow_ma, exit_ma | "MA5上穿MA20买入" |
| `squeeze_breakout` | 收缩突破 | timeframe, direction, hold_bars | "H1收缩后突破买入" |
| `state_hex_direction` | State Hex方向 | state_pattern, direction | "D1=6且H1=4时做多" |
| `bollinger_bounce` | 布林带反弹 | bb_period, std_dev | "布林带下轨买入" |
| `pivot_reversal` | 枢轴反转 | pivot_type, lookback | "D1枢轴支撑买入" |
| `multi_timeframe` | 多周期共振 | h1_state, h4_state, d1_state | "H4/H1共振做多" |

#### 3.3.3 各Agent职责

**Research Agent (研究Agent)**
```python
class ResearchAgent:
    """研究Agent - 策略研发与模式识别"""
    
    responsibilities = [
        "strategy_mining",      # State Hex策略挖掘
        "pattern_recognition",  # 多周期共振模式识别
        "reification_detection", # 复现检测 (历史模式重现)
        "data_analysis",        # 数据质量与异常分析
    ]
    
    outputs = [
        "StrategyReport",       # 策略研究报告
        "PatternAlert",         # 模式发现告警
        "ReificationSignal",    # 复现匹配信号
    ]
```

**Execution Agent (执行Agent)**
```python
class ExecutionAgent:
    """执行Agent - 订单执行与绩效追踪"""
    
    responsibilities = [
        "signal_processing",   # 信号过滤与优先级排序
        "order_execution",      # MT5订单发送
        "position_management",  # 仓位监控与调整
        "performance_tracking", # 实时绩效统计
        "deviation_detection", # 回测-实盘偏差检测
    ]
    
    outputs = [
        "OrderResult",         # 订单执行结果
        "PositionUpdate",      # 仓位更新
        "PerformanceReport",   # 绩效报告
        "DeviationAlert",      # 偏差告警
    ]
```

```python
class RiskAgent:
```

## 6. 风险与缓解

| 风险类型 | 风险描述 | 影响 | 缓解措施 |
|----------|----------|------|----------|
| **技术风险** | 回测过拟合 | 高 | Walk-Forward强制、样本量门槛 |
| **技术风险** | 实盘偏差大 | 高 | 模拟盘过渡、偏差监控 |
| **技术风险** | MT5连接中断 | 中 | 自动重连、备用数据源 |
| **市场风险** | 极端行情失效 | 高 | 熔断机制、风控审核 |
| **操作风险** | 错误交易 | 高 | 二次确认、限额控制 |
| **数据风险** | 历史数据不完整 | 中 | 数据质量检查、填充规则 |
