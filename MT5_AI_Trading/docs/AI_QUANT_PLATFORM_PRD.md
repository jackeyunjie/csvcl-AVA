# AI量化赋能平台 — 产品需求文档 (PRD)

> 版本: v1.0  
> 日期: 2026-06-06  
> 目标: 站在开源项目肩膀上，构建面向个人交易者的AI策略生成-观察-实施闭环平台

---

## 1. 项目背景与目标

### 1.1 现状痛点

当前MT5_AI_Trading项目已积累大量实证基础设施：

- **State Hex编码系统**: P107标准，0-F十六进制状态，支持多周期组合分析
- **回测平台**: 数据层→计算层→策略层→执行层→展示层五层架构
- **策略挖掘**: StrategyMiner支持state_pattern × direction × hold_bars组合搜索
- **观察数据库**: DuckDB持久化关键观察，支持复现检测（70%/80%阈值提醒）
- **多Agent辩论**: H1/M15收缩突破、State策略挖掘并行验证

**但缺失面向"用户赋能"的顶层产品化能力** — 系统目前仅供内部研究使用，无法让普通交易者：
- 用自然语言描述想法 → 自动生成可回测策略
- 观察市场状态 → 获得AI生成的策略建议
- 一键将验证通过的策略部署到MT5模拟/实盘

### 1.2 目标愿景

> **"让每位交易者都能拥有AI量化研究员的能力"**

构建一个AI量化平台，用户只需：
1. **描述** — 用自然语言描述交易想法（"我想在黄金H1收缩后做多"）
2. **生成** — AI自动生成策略代码 + 回测配置
3. **验证** — 平台自动运行Walk-Forward回测，输出绩效报告
4. **观察** — 平台持续扫描市场，当策略条件满足时推送提醒
5. **实施** — 一键部署到MT5模拟盘，验证后再上实盘

## 3. 核心功能需求

### 3.1 策略生成引擎 (Strategy Generator)

**输入**: 自然语言描述 + 可选参数（品种、周期、方向）  
**输出**: 完整策略代码 + 回测配置 + 策略说明文档

**示例交互**:
```
用户: "黄金H1出现收缩突破时做多，持仓5根K线，止损前低"

AI生成:
- 策略代码 (Python/State Hex)
- 回测配置 (品种=XAUUSD, 周期=H1, 时间范围=2020-2025)
- Walk-Forward参数 (训练60%, 验证20%, 测试20%)
- 预期输出: 胜率、盈亏比、最大回撤、夏普比率
```

**技术实现**: 
- LLM (GPT-4/Claude) 解析自然语言 → 提取策略要素
- 模板引擎将要素填充到策略模板（State Hex策略、收缩突破策略、均线策略等）
- 代码生成后自动语法检查 + 回测验证

### 3.2 自动回测验证 (Auto Backtest)

**输入**: 策略代码 + 回测配置  
**输出**: 标准化绩效报告

**报告内容**:
- 总体绩效: 累计收益、年化收益、夏普比率、最大回撤
- Walk-Forward三段: Train/Validation/Test分别统计
- State-Regime分析: 按W1|MN1状态分组展示胜率
- 交易明细: 每笔交易的入场/出场时间、盈亏、持仓周期
- 可视化图表: 净值曲线、回撤曲线、月度收益热力图

**关键要求**:
- 无未来函数验证（一票否决）
- MT5实际点差模拟（非固定成本）
- 支持多品种、多周期批量回测
- 结果可导出SQX格式供MT5导入

### 3.3 市场观察站 (Market Observer)

**功能**: 7×24小时扫描市场，检测策略触发条件

**观察维度**:
- **State Hex组合**: H1视角下D1/H4的0,8组合变化
- **收缩突破**: M15/H1周期收缩后的突破信号
- **复现检测**: 当前市场状态与历史关键观察的匹配度（70%/80%阈值）
- **多品种监控**: XAUUSD, EURUSD, GBPUSD, US30, GER40, DXY等

**输出**:
- 实时状态面板（Web/桌面）
- 条件触发时推送通知（Telegram/邮件/飞书）
- 每日观察报告（Markdown/HTML）

### 3.4 策略实施通道 (Strategy Deployment)

**模拟盘**:
- 一键部署到MT5模拟账户
- 自动记录模拟交易结果
- 与回测结果对比，检测"回测→模拟"偏差

**实盘**:
- 风控层审核（最大仓位、单日止损、品种分散）
- 信号→MT5 EA自动执行
- 实盘绩效追踪与回测预期对比

### 3.5 策略市场 (Strategy Marketplace) [未来]

- 策略开发者发布"策略卡片"
- 其他用户订阅使用，按绩效付费
- 平台抽成 + 开发者分成

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标 |
|------|------|
| 策略生成响应 | < 30秒（LLM + 代码生成） |
| 单策略回测（5年H1数据） | < 2分钟 |
| 市场扫描频率 | H1周期每15分钟，M15周期每5分钟 |
| 并发回测 | 支持10个策略同时回测 |
| 数据存储 | 支持100个品种×5个周期×10年历史 |

### 4.2 可靠性要求

- 回测结果100%可复现（固定随机种子）
- 数据源故障自动切换备用源
- 系统崩溃后状态恢复（持仓、待执行订单）
- 实盘交易异常自动暂停并告警

### 4.3 安全要求

- API Key加密存储（MT5密码、券商API）
- 实盘交易需二次确认
- 策略代码沙箱执行（防止恶意代码）
- 用户数据隔离（多租户架构）

## 5. 竞品对标

| 平台 | 我们的优势 | 我们的劣势 |
|------|-----------|-----------|
| **Microsoft Qlib** | 更贴近MT5实盘，支持外汇/CFD | 机器学习深度不如Qlib |
| **FinRL/FinRL-X** | 策略生成更灵活（自然语言输入） | RL研究深度不如FinRL |
| **VectorBT** | 端到端闭环（生成→回测→部署） | 纯回测性能不如VBT |
| **QuantConnect Lean** | 更轻量，个人交易者友好 | 云端基础设施不如QC |
| **TradingView** | 可部署到MT5实盘 | 社区和图表不如TV |
| **NautilusTrader** | 策略生成AI化 | 执行引擎性能不如NT |

**差异化定位**: 
> "唯一支持'自然语言→策略生成→Walk-Forward回测→MT5实盘部署'完整闭环的开源平台"

## 6. 里程碑规划

### Phase 1: 内核强化（4-6周）
- [ ] 验证merge_asof无未来函数
- [ ] MT5实际点差接入
- [ ] v5单元测试覆盖>80%
- [ ] 4周模拟盘观察

### Phase 2: AI策略生成（6-8周）
- [ ] 自然语言→策略要素提取（LLM）
- [ ] 策略模板库（State Hex、收缩突破、均线等10+模板）
- [ ] 代码生成 + 自动语法检查
- [ ] 生成策略自动回测验证

### Phase 3: 观察站产品化（4-6周）
- [ ] Web状态面板（实时State Hex、收缩突破）
- [ ] 多通道通知（Telegram/飞书/邮件）
- [ ] 观察报告自动生成
- [ ] 复现Agent集成到观察站

### Phase 4: 部署通道（4-6周）
- [ ] MT5模拟盘一键部署
- [ ] 实盘风控层
- [ ] 绩效追踪与偏差检测

### Phase 5: 社区与商业化（未来）
- [ ] 策略市场
- [ ] 付费订阅
- [ ] 多语言支持

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| LLM生成策略质量不稳定 | 高 | 模板约束 + 回测验证门槛 |
| 回测过拟合 | 高 | Walk-Forward强制 + 样本量门槛 |
| 实盘与回测偏差大 | 高 | 模拟盘过渡 + 偏差监控 |
| MT5连接不稳定 | 中 | 重连机制 + 备用数据通道 |
| 数据源成本 | 中 | 优先免费源（Yahoo/MT5历史） |
