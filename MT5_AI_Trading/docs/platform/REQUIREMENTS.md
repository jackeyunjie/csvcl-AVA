# AI量化平台需求文档 (PRD)

> 版本: v1.0 | 日期: 2026-06-06 | 状态: 草案
>
> 本文档定义"AI赋能策略生成、总结、观察、实施"量化平台的功能需求。

---

## 一、产品愿景

### 1.1 一句话描述

**让用户可以独立部署一个AI量化工作站**, 通过自然语言描述交易想法→AI自动挖掘策略→多Agent辩论验证→模拟盘观察→实盘执行的完整闭环。

### 1.2 目标用户

| 用户类型 | 特征 | 核心诉求 |
|----------|------|----------|
| **独立交易员** (你) | 有自己的交易经验, 使用MT4/MT5 | 把直觉量化为可验证的策略, AI辅助而非替代 |
| **量化学习者** | 懂Python/数据分析, 想入门量化 | 降低策略开发门槛, 看到AI怎么思考和决策 |

### 1.3 核心差异化

| 维度 | 现有平台 | 我们的平台 |
|------|----------|-----------|
| **策略来源** | 用户自己写代码/规则 | AI自动挖掘 + 用户经验注入 |
| **验证方式** | 单次回测 | **多Agent辩论** (不同周期/不同视角互相质疑) |
| **策略进化** | 用户手动调参 | AI持续监控策略衰减, 自动触发重优化 |
| **可观测性** | 只看盈亏 | State视角 — 能看到市场"状态"和策略"匹配度" |
| **部署门槛** | 需要服务器/运维 | 单机即可 (Windows), MT5直连 |

## 二、故事线 (User Journey)

```
用户: "我觉得DAX在M15收缩后做空, 用H1趋势过滤, 止损80点, 盈亏比3:1"

AI平台:
  Step 1 — 策略生成 (R&D Agent)
    → 自动将自然语言翻译为可执行策略
    → 自动获取GER40的M15/H1/H4数据
    → 生成squeeze_score计算 + 突破检测 + 出场规则代码
    → 当前进度: 已有v5引擎 + M15 Phase1脚本

  Step 2 — 回测验证
    → Walk-Forward三段回测 (Train/Val/Test)
    → 输出: 胜率、盈亏比、净期望、最大回撤
    → 当前进度: v5已实现

  Step 3 — 多Agent辩论
    → Agent_H1_Squeeze: "H1视角下这个策略Test期望+0.299%, 统计可信"
    → Agent_M15_Squeeze: "M15收缩密度18.7%, 共振25.2%, 值得加入"
    → Agent_State_Short: "State策略short方向胜率92.8%, 也支持做空"
    → 辩论合成: 三方数据一致支持 → 进入模拟盘
    → 当前进度: AGENTS.md已定义辩论规则, 待实现自动化

  Step 4 — 模拟盘观察
    → 每日自动扫描14个品种的收缩状态
    → 发现setup后推送通知
    → 记录每笔模拟交易, 生成日报/周报
    → 当前进度: run_v5_simulation.py + daily_summary 已实现

  Step 5 — 策略健康监控
    → 模拟盘运行4周后, AI自动评估
    → 如果实盘表现偏离回测>30%, 触发告警
    → 如果市场状态变化, 建议重新优化参数
    → 当前进度: 待实现

  Step 6 — 实盘执行 (远期)
    → 一键将验证通过的策略部署到MT5
    → 风险控制: 仓位上限、日内止损、最大回撤熔断
    → 当前进度: 暂不实现 (先用模拟盘积累信心)
```

## 三、功能需求

### F1: 策略生成引擎 (R&D Layer)

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F1.1 | 自然语言→策略骨架 | 用户描述:"M15收缩+H1趋势过滤+3:1盈亏比" → 自动生成策略代码模板 | P1 |
| F1.2 | 自动数据获取 | 根据策略需要的品种/周期, 从MT5自动拉取数据 | P1 |
| F1.3 | 参数扫描 | 对关键参数(max_adx, min_range, cooldown)自动网格搜索 | P1 |
| F1.4 | 因子挖掘 (远期) | 基于RD-Agent模式: LLM提假设→自动实现→回测→迭代 | P3 |

**参考项目**: 微软 RD-Agent (R&D loop: Hypothesis → Implement → Test → Feedback)

---

### F2: 多Agent辩论系统 (Debate Layer)

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F2.1 | Agent注册 | 每个Agent有自己的周期/品种/策略, 产出标准报告 | P0 |
| F2.2 | 辩论调度 | 按资产类别组合Agent (如: 股指=H1+M15, 股票=D1+W1) | P1 |
| F2.3 | 共识合成 | 自动对比各Agent的Walk-Forward三段数据, 生成共识结论 | P1 |
| F2.4 | 辩论过程可视化 | 用户可以看到每个Agent的论点和数据依据 | P2 |
| F2.5 | 异议处理 | 如果Agent之间结论冲突, 标记为"需要人工判断" | P1 |

**参考项目**: ContestTrade (内部竞赛机制), QuantAgents (多Agent会议), OpenFinClaw (Agent Debate Arena)

---

### F3: 回测与验证 (Backtest Layer)

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F3.1 | Walk-Forward回测 | Train/Validation/Test三段, 防过拟合 | P0 (已实现v5) |
| F3.2 | 交易成本建模 | MT5实际点差+手续费, 非固定值 | P1 |
| F3.3 | as-of时间戳对齐 | `pd.merge_asof` 防未来函数泄漏 | P0 (v3已修复) |
| F3.4 | 多出场规则对比 | fixed_hold / structure_stop / 1r_partial / trailing | P1 (v5已实现) |
| F3.5 | 蒙特卡洛模拟 | 对回测结果做随机扰动, 评估过拟合风险 | P2 |
| F3.6 | 样本量充足性检验 | 自动判断Test段样本是否够统计显著 (n>50) | P1 |

**参考项目**: 微软 Qlib (回测引擎+交易成本+策略评估), vnpy (CtaBacktesting)

---

### F4: 模拟盘观察 (Observation Layer)

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F4.1 | 实时状态扫描 | 每15分钟扫描14品种的收缩/趋势状态 | P0 (已实现) |
| F4.2 | 信号推送 | 发现squeeze setup时推送通知 (Telegram/钉钉) | P2 |
| F4.3 | 日报生成 | 每日自动汇总: 扫描状态、历史信号、市场环境 | P1 (已实现) |
| F4.4 | 周报/月报 | 含绩效归因: 哪些品种贡献利润, 哪些在亏损 | P2 |
| F4.5 | 策略健康度仪表盘 | 绿/黄/红: 实盘vs回测偏差、参数稳定性 | P1 |

**参考项目**: Freqtrade (Telegram bot + Web UI 监控), vnpy (实时行情+持仓监控)

---

### F5: 策略生命周期管理 (Lifecycle Layer)

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F5.1 | 策略版本管理 | 每个策略有版本号, 记录参数/数据窗口/回测结果 | P1 |
| F5.2 | 衰减检测 | 模拟盘表现偏离回测超阈值 → 自动告警 | P2 |
| F5.3 | 自动重优化 | 检测到衰减后, 自动扩大数据窗口重新扫描参数 | P3 |
| F5.4 | 策略退役 | 连续N周负期望 → 自动标记为"已退役" | P2 |

---

### F6: 用户界面

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| F6.1 | CLI命令行 | `pipeform scan` / `pipeform backtest` / `pipeform debate` | P1 |
| F6.2 | Web仪表盘 | 策略列表、模拟盘状态、日报/周报、Agent辩论记录 | P2 |
| F6.3 | 自然语言交互 | 直接在Web界面输入交易想法, AI返回策略建议 | P3 |

**参考项目**: OpenFinClaw (CLI + Natural Language → Strategy), Freqtrade (CLI + Web UI)

## 四、非功能需求

### N1: 性能
- M15数据扫描 (14品种×365天) ≤ 5分钟
- 参数敏感性分析 (60组) ≤ 30分钟
- 模拟盘扫描 (14品种) ≤ 10秒

### N2: 可靠性
- MT5断连自动重试, 重试超过3次告警
- 回测结果可复现 (固定随机种子)
- 数据缓存: 同一天不重复从MT5拉取

### N3: 安全性
- API密钥存 `.env`, 不提交git
- MT5交易密码不存磁盘明文

### N4: 可维护性
- 每个Agent独立文件, 统一接口
- 报告格式标准化 (Markdown + 结构化JSON)
- 代码注释使用中文

### N5: 兼容性
- Windows 10/11 + Python 3.12
- MT5终端 (AvaTrade版本)
- RDS MySQL 8.0 (阿里云, 已在用)
