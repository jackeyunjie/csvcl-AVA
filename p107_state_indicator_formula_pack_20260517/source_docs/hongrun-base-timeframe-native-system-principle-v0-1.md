# 弘运基准周期原生系统独立原则 v0.1

## 一句话

D1、H1、M15、M5 等不是同一个系统的缩放页面，而是不同资金周期、不同交易者类型、不同可见信息边界下的独立基准系统。

因此，H1 不是 D1 的下钻，M15 也不是 H1 的细化页面。每一个基准周期都必须从自己的价格路径出发，生成自己的高周期上下文、状态池、标签审计和验证流水线。

## 为什么必须独立

不同基准周期面对的是不同问题：

| 基准周期 | 典型用户 | 核心问题 | 高周期上下文 |
|---|---|---|---|
| D1 | 中长线投资者、日线波段研究者 | 今天是否值得进入观察池，是否处于大周期支持环境 | W1 / MN1 |
| H1 | 短线交易者、盘中研究者 | 当前小时级结构是否和高周期状态匹配，是否需要等待或回避 | H1 原生拟合的 D1 / W1 / MN1 |
| M15 | 更短线、期货/大宗/外汇/加密等盘中系统 | 低周期动作是否只发生在合适的高周期环境里 | M15 原生拟合的 H1 / D1 / W1 / MN1 |

如果把 D1 结论直接套到 H1 或 M15，会出现两个问题：

1. **信息边界错误**：H1 某一根 K 线发生时，日线、周线、月线可能尚未闭合，不能偷看未来才定型的状态。
2. **用户目标错误**：D1 系统服务中长期观察，H1/M15 系统服务短线节奏，两者的策略原语、持有周期、风险约束和样本密度都不同。

## 工程铁律

每一个基准周期必须独立具备以下资产：

```text
state_timeframe_<base_tf>          -- 该基准周期自己的价格和状态事实源
state_<base_tf>_multitf_asof       -- 该基准周期自己的多周期 as-of 宽表
state_regime_pool_<base_tf>        -- 该基准周期自己的状态组合样本池
label_audit_<base_tf>              -- 该基准周期自己的 forward label 可用性审计
verify_<base_tf>_*                 -- 该基准周期自己的泄漏、覆盖、边界检查
```

其中 H1/M15 的高周期上下文必须由本基准周期的价格路径聚合或拟合得到，不允许直接复用上一级系统的裁决表冒充“当时可见环境”。

最低 as-of 条件：

```text
high_tf_state_date < base_event_timestamp_or_session_date
same_session_postclose_used = 0
strict_prior_violation_count = 0
```

对于盘中系统，尤其要避免“上午读取当天收盘后才定型的 D1 状态”。

## 与 P17 的关系

P17 State-Regime Walk-Forward 仍然是主裁决，但它必须落在每个基准周期自己的状态池上。

例如：

- D1 系统：把不同年份里相同 W1/MN1 状态组合下的 D1 样本拼成状态池。
- H1 系统：把不同年份里相同 H1 原生高周期状态组合下的 H1 样本拼成状态池。
- M15 系统：把不同年份里相同 M15 原生高周期状态组合下的 M15 样本拼成状态池。

Time-Drift Audit 仍然只是时间漂移旁路审计，不替代状态组合窗口主裁决。

## 对国际大宗和更细周期的适用

未来如果国际大宗、期货、外汇、加密等品种下沉到 H1、M15 或更细周期，仍然遵守同一原则：

- 先建立该基准周期自己的原始数据域和交易时段口径。
- 再生成该基准周期自己的高周期状态上下文。
- 再抽取该基准周期自己的状态池。
- 再做 label 可用性审计。
- 最后才进入 P17 状态组合窗口验证。

不能因为已有 D1 系统或已有某个市场的 H1 系统，就跳过新市场、新周期的独立审计。

## 推荐表述

推荐：

- “H1 是短线用户的基准周期原生系统，带有 H1 当时可见的 D1/W1/MN1 上下文。”
- “M15 是更细周期的独立系统，需要重新完成 as-of、状态池和 label 审计。”
- “D1、H1、M15 对应不同交易者和资金周期，不是同一个页面的缩放。”

避免：

- “H1 是 D1 的下钻。”
- “M15 是 H1 的细化页面。”
- “日线系统验证通过，所以小时线可以直接使用。”
- “把 D1 的 P17 结论套到 H1/M15。”

## 当前落地状态

- A 股 D1：已有日线多周期时间戳对齐系统，服务中长线和日线观察。
- 美股 H1：正在建立 H1 原生全周期对齐系统，从 H1 price path 聚合/拟合 D1/W1/MN1 上下文，目标是短线系统的独立底座。
- 未来国际大宗 H1/M15：按本原则另起独立系统，不从 A 股 D1 或美股 H1 直接继承裁决。
