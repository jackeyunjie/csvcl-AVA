# ACD 逻辑交易

## 一句话定义

ACD 是由 Mark Fisher 提出的一种日内交易框架，以开盘区间（A 点）为基准，通过价格突破 A 点上方/下方特定阈值来判定多空方向，并结合 B/C/D 点管理入场、止损与目标。

## 为什么重要

ACD 将开盘后的价格行为结构化为一组清晰的规则，帮助交易者：

1. 识别日内多空主导方向
2. 过滤开盘噪音，等待真正有效的突破
3. 以开盘区间为核心，建立一致性的风险管理

## 具体内容

### ACD 核心概念

| 点/区域 | 含义 | 交易意义 |
|---------|------|----------|
| **A 点** | 开盘区间的最高点（A+）和最低点（A-） | 多空分水线，突破 A+ 看多，跌破 A- 看空 |
| **B 点** | 开盘区间中点，通常为 (A+ + A-) / 2 | 判断强弱：价格收于 B 点上方偏强，下方偏弱 |
| **C 点** | 突破 A 点后回踩并再次突破的确认点 | 过滤假突破，提供更安全入场 |
| **D 点** | 反向突破点，通常用于平仓或反手 | 趋势失败信号 |

### 典型规则

1. **计算开盘区间**：通常取开盘后 20~30 分钟的高低点。
2. **确定 A+/A-**：区间高点为 A+，低点为 A-。
3. **添加缓冲**：A+ + 缓冲 = 做多触发线；A- - 缓冲 = 做空触发线。
4. **等待突破并确认**：价格触发后，回踩不回到区间内，再次突破时入场。
5. **止损**：设在 A-（多单）或 A+（空单）附近。

### 与 ORB 的对比

| 维度 | ACD | ORB |
|------|-----|-----|
| 核心 | 开盘区间 + 固定缓冲点 | 开盘区间 + 回踩确认 |
| 触发 | 突破 A+/A- 并触及 C 点 | 收盘价突破区间并回踩再突破 |
| 程序实现 | 需明确定义 A/B/C/D 阈值 | 更通用，适合状态机实现 |
| 适用性 | 期货、股票日内 | 外汇、期货、股票多市场 |

### 与本项目的结合点

- 若 MT4/MT5 端输出 ACD 相关状态码（如 A+ 突破、C 点确认），CSV 处理脚本可据此着色、统计、生成邮件警报。
- 颜色规则可沿用本项目：红色表示多向突破，绿色表示空向突破。
