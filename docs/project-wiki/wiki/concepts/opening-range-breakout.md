# 开盘区间突破（Opening Range Breakout, ORB）

## 一句话定义

开盘区间突破（ORB）是一种日内交易策略，通过捕捉交易时段最初一段固定时间（如 15/30 分钟）内形成的高低价区间，并在价格有效突破该区间边界时跟随趋势入场。

## 为什么重要

早盘（或特定交易时段开盘）往往集中了隔夜信息、经济数据和机构订单的首次反应，形成当日关键的高低边界。突破这些边界通常意味着供需力量失衡，可能启动一段持续趋势。ORB 提供了客观、可量化的入场框架，避免主观猜测。

## 具体内容

### 区间计算方法

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| 双 K 线区间 | 取昨日最后一根 K 线 + 今日开盘第一根 K 线的高低极值 | 快速确定隔夜到开盘的扩展区间 |
| 固定分钟区间 | 开盘后 N 分钟内跟踪最高价/最低价 | 标准 ORB，反映开盘初期真实波动 |

### 三步确认机制

1. **突破（Break）**：收盘价收于开盘区间之外
2. **回踩（Retest）**：价格回撤至区间边界附近
3. **再突破（Confirm）**：收盘价再次收于区间外，确认入场

> 三步确认可有效过滤“假突破”，避免在流动性枯竭时追单。

### 典型参数

- 区间时长：`RangeMinutes` = 15~30 分钟
- ATR 周期：14
- ATR 倍数：1.5（用于动态止损/止盈）

### 止损与止盈

- 止损通常设在区间另一侧边界内侧
- 止盈可基于 ATR 倍数或固定风险回报比（如 1:2 / 1:3）

### MQL5 实现要点

- 使用 `SymbolInfoSessionTrade()` 获取交易时段开始时间
- 使用 `iBarShift()` 定位开盘后的第一根 K 线
- 使用状态机管理 Capture → Break → Retest → Done 流程
- 每日午夜重置状态，避免跨日逻辑 Bug

## 与 ACD 逻辑的关系

- ACD 的 **A 点** 本质上就是开盘区间的上下边界。
- ORB 的“回踩确认”可视为 ACD 中 **C 点** 的变体。
- ACD 更强调特定点差（如 +/- 0.5 ATR）和明确的 A/B/C/D 四个交易区域；ORB 更通用、更程序化。
