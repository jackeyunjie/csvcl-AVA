# 概念：Vegas 隧道交易系统

## 一句话定义

Vegas 隧道交易系统是一种**多周期均线趋势跟踪模型**：用 EMA 144/169 构成中短期“隧道”作为多空分水岭与动态支撑阻力，EMA 576/676 判断长期趋势方向，EMA12 过滤假突破，Volume Oscillator 确认突破有效性。

## 核心逻辑

### 1. 均线系统分层

- **短期过滤**：EMA 12 代表短期动量，只有在隧道方向一致时才允许入场。
- **中短期趋势/支撑阻力**：EMA 144/169 组成 Vegas 隧道，是多空分水岭和回调入场区域。
- **长期趋势**：EMA 576/676 是四倍于隧道的均线，决定大方向；交易方向必须与其一致。

### 2. 趋势确认与过滤

- 隧道与长期均线形成同向排列，确认趋势方向。
- EMA12 位于隧道同侧，过滤反向假突破。
- Volume Oscillator 突破零线用于确认突破是否由大资金推动。

### 3. 分层止损

- **第一层**：Volume Osc 确认突破有效性 → 有效则止损。
- **第二层**：24 小时时间窗口 + 加速判断 → 防止毛刺触发。
- **第三层**：EMA 676 硬止损 → 任何周期都遵守的纪律线。

## 关键要素

| 要素 | 要求 | 说明 |
|------|------|------|
| 大趋势均线 | EMA 576/676 | 决定多空方向，交易必须顺势 |
| Vegas 隧道 | EMA 144/169 | 多空分水岭、动态支撑阻力、回调入场区 |
| 过滤线 | EMA 12 | 必须位于隧道同侧 |
| 量能指标 | Volume Osc(1,14,144) | 确认突破真假；<1H 可用 89 |
| 入场时机 | 价格回踩隧道 | 可设置价格区间分批挂单 |
| 止损纪律 | EMA 676 跌破必出 | 任何周期的最后防线 |

## 常见错误

- 在均线缠绕、方向不明的行情中强行交易。
- 价格刚跌破隧道就立即止损，未经过 Volume Osc / 24 小时观察流程。
- 忽视 EMA 676 硬止损，导致小亏变大亏。
- 在弱势趋势中频繁加仓超过 2 次。

## 程序化实现方向

- **Python**：使用 `pandas-ta` 或 `talib` 计算 EMA 与 Volume Oscillator，在 Backtrader / VectorBT 中实现回调入场与分层止损。
- **MQL5/MQL4**：调用 `iMA` 与自定义 Volume Osc 指标，输出状态 CSV 供本项目着色流程使用。
