# SOP：技术面分析 → 雷达/因子图 → 飞书报告

**版本**：v1.2  
**适用场景**：XAUUSD 等多品种多时间框架技术面分析、入场时机判断、可视化报告输出  
**目标**：建立从截图/源码 → 量化评分 → **等待“桃子成熟”的安全入场点** → 可视化报告 → 飞书交付的标准化流程

## 二、详细步骤

### Step 1：输入采集

**输入**：
- 目标品种的 M15 / M30 / H1 / H4 截图（至少 2 个周期）
- 截图中使用的指标名称与参数
- 分析目标（如：判断反转突破概率、验证指标效果等）

**操作**：
1. 截图保存到项目目录 `uploads/` 或会话上传区。
2. 记录每个截图的周期、品种、时间范围。
3. 若涉及自定义指标，收集 `.mq4` / `.mq5` 源码。

**输出**：
- 原始截图文件
- `input_meta.md`：记录品种、周期、指标、目标

**负责**：用户 / 数据采集人员

---

### Step 2：指标准备（如涉及 MT4→MT5 改造）

**输入**：MT4 指标源码（`.mq4`）

**操作**：
1. 读取 `.mq4` 源码，识别关键逻辑：
   - `init()` / `start()` 函数
   - 缓冲区声明与 `SetIndexBuffer`
   - 使用的内置函数（`iMA`, `iRSI`, `iMAOnArray`, `iRSIOnArray` 等）
   - 输入参数与颜色常量
2. 转换为 MQL5：
   - `init()` → `OnInit()`
   - `start()` → `OnCalculate()`
   - `extern` → `input`
   - `IndicatorBuffers()` 删除，改用 `#property indicator_buffers N`
   - `SetIndexBuffer(..., INDICATOR_DATA)`
   - `PlotIndexSetInteger` / `PlotIndexSetDouble` 设置样式
   - 颜色常量改为 `clr*` 或裸 Web 颜色名
3. 使用句柄链路替换 `iMAOnArray` / `iRSIOnArray`：
   ```
   iMA(价格) → iRSI(MA句柄) → iMA(RSI句柄)
   ```
4. 编译验证（MetaEditor 5）。

**输出**：
- `MT5/Indicators/指标名.mq5`
- 编译通过截图或日志

**负责**：AI Agent / 技术开发

---

### Step 3：图表定性分析

**输入**：截图 + 已改造的指标

**操作**：
1. 识别主图结构：
   - 趋势方向（多/空/震荡）
   - 关键支撑/阻力位置
   - K 线形态（反转形态、突破、假突破等）
2. 识别副图指标信号：
   - RSIOMA：金叉/死叉、拐头、背离
   - ADX：趋势强度
   - KaufmanBandsWidth / Sq 系列：波动率、收缩/扩张
3. 多周期对比：
   - M15 与 M30 结构是否一致
   - 小周期信号是否被大周期支撑/压制
4. 标记关键区域：
   - 收缩区域
   - 反转突破点
   - 假跌破/假突破位置

**输出**：
- 定性分析笔记（文字描述）
- 在截图上标注关键区域（可选）

**负责**：AI Agent / 分析师

---

### Step 4：量化建模

**输入**：定性分析结论

**操作**：
1. 确定评分维度（雷达图）：
   - 趋势强度
   - 动量/RSIOMA
   - SR 结构
   - 多周期共振
   - 波动率收缩
   - 反转确认
   - 风险可控性
2. 确定影响因子（因子图）：
   - 正向因子：H4 SR 支撑、H1 SR 收缩、枢轴收敛、RSIOMA 金叉、ADX 配合、假跌破收回
   - 负向因子：上方阻力、宏观不确定性、量能不足等
3. 建立评分公式：
   ```
   Reversal_Score = Σ(正向因子 × 权重 × 置信度) - Σ(负向因子 × 权重 × 置信度)
   Final_Score = 5 + (Reversal_Score / Max_Possible_Score) × 5
   ```
4. 设定阈值规则：
   - ≥ 8.5：高优先级
   - 7.0 ~ 8.4：中等偏强，继续观察
   - 5.0 ~ 6.9：结构模糊
   - < 5.0：偏弱，放弃

**输出**：
- 维度得分表
- 因子得分表
- 业务评分公式与阈值

**负责**：AI Agent / 量化建模

---

### Step 5：等待入场时机（桃子成熟）

> **核心原则**：分析完成 ≠ 立刻入场。入场点是等出来的，要等到“桃子成熟”——即反转结构确认、风险可控、盈亏比合理的时刻。

#### 5.1 “桃子成熟”的判定标准

入场前必须同时满足以下 **5 个成熟度条件**：

| 成熟度条件 | 具体标准 | 为何重要 |
|-----------|---------|---------|
| **结构成熟** | H4 SR 支撑被测试 1~3 次，价格未有效跌破 | 确认大周期支撑有效 |
| **收缩成熟** | H1 SR 带 / 1日/3日枢轴明显收窄，波动率下降 | 能量蓄积，突破后方向更清晰 |
| **动量成熟** | RSIOMA 低位金叉后，黄线（MaRsioma）持续向上 | 动能由空转多，不是假金叉 |
| **确认 K 线** | 出现一根收盘站稳 H4 SR 上沿或 H1 SR 带上沿的阳线 | 价格行为确认突破 |
| **风险可控** | 止损位明确（H4 SR 下沿下方 1~2 ATR），单笔风险 ≤ 账户 2% | 即使看错也有退路 |

#### 5.2 入场触发类型（按激进程度）

| 触发类型 | 条件 | 适用场景 | 风险等级 |
|---------|------|---------|---------|
| **激进型** | RSIOMA 金叉瞬间，价格收在 H1 SR 带上沿 | 结构非常清晰、时间紧迫 | 高 |
| **稳健型** | 金叉后回踩 H1 SR 带上沿不破，再次出现阳线 | 最常见的“桃子成熟”入场点 | 中 |
| **保守型** | 价格突破并站稳 H4 SR 上沿，且 H1 周期同步创新高 | 确认大周期突破有效 | 低 |

**推荐默认使用稳健型触发**。

#### 5.3 入场前强制检查清单

在点击下单前，必须逐项确认：

- [ ] 雷达图综合得分 ≥ 7.0
- [ ] 因子图净效应 > 0（正向因子占优）
- [ ] H4 周期方向偏多或至少不空
- [ ] 入场 K 线不是长上影线或十字星
- [ ] 止损位距离入场价 ≤ 1.5 倍目标位距离（风险回报比 ≥ 1:1.5）
- [ ] 未来 4 小时内无重大数据/事件（如非农、美联储讲话、地缘冲突）
- [ ] 仓位大小已根据止损距离计算好

#### 5.4 “桃子未熟”的信号（应继续等待）

出现以下任一情况，说明入场时机不成熟，继续观望：

- RSIOMA 金叉后很快死叉（假金叉）
- 价格突破 H1 SR 上沿但成交量/动量未跟上
- 大周期 H4 仍在空头排列，小周期反弹只是回调
- 价格仍在 H4 SR 带内部震荡，没有明确方向
- 距离上方强阻力太近，目标空间不足

#### 5.5 失效条件（应放弃该结构）

出现以下任一情况，说明结构失效，不再跟踪：

- 价格收盘跌破 H4 SR 下沿，且 2 根 K 线未收回
- RSIOMA 黄线拐头向下并下穿青线
- 出现重大利空消息导致趋势突变
- 连续 3 次测试 H4 SR 下沿后被跌破

#### 5.6 仓位与止损规则

```
止损位 = H4 SR 下沿 - 1~2 ATR
目标位1 = 最近 H4 阻力上沿
目标位2 = 1.5~2 倍止损距离的波段目标
仓位大小 = (账户风险金额) / (止损点数 × 每点价值)
```

**示例**：
- 账户 10,000 USD，单笔风险 2% = 200 USD
- XAUUSD 止损 5 USD/盎司，止损距离 8 美元
- 仓位 = 200 / (8 × 1) = 0.25 手

**输出**：
- `entry_plan.md`：明确入场触发、止损、目标、仓位

**负责**：AI Agent / 交易员

### Step 6：可视化

**输入**：维度得分、因子得分、多周期评分、入场计划

**操作**：
1. 使用 Python + Plotly 生成：
   - 雷达图（当前 / 理想 / 弱势 三条线）
   - 因子贡献条形图
   - 多时间框架热力图
2. 导出 PNG 图片：
   ```python
   fig.write_image("reports/radar_chart.png", scale=2)
   fig.write_image("reports/factor_chart.png", scale=2)
   fig.write_image("reports/heatmap_chart.png", scale=2)
   ```
3. 生成交互式 HTML 报告：
   ```python
   fig.to_html(full_html=True, include_plotlyjs='cdn')
   ```

**输出**：
- `reports/radar_chart.png`
- `reports/factor_chart.png`
- `reports/heatmap_chart.png`
- `reports/XXX_Technical_Radar_Report.html`

**负责**：AI Agent

**输出**：

### Step 8：飞书交付

**输入**：Markdown 报告、PNG 图片、HTML 报告

**操作**：
1. 准备飞书文档 Markdown 内容（`reports/feishu_doc_content.md`）。
2. 登录飞书授权：
   ```bash
   lark-cli auth login --scope "docx:document docx:document:write_only docs:document.media:upload docs:document.content:read"
   ```
3. 创建飞书文档：
   ```bash
   lark-cli docs +create --title "报告标题" --markdown (Get-Content -Raw report.md)
   ```
4. 记录返回的 `doc_id` 和 `doc_url`。
5. 插入图片到文档末尾：
   ```bash
   lark-cli docs +media-insert --doc doxcnXXX --file ./radar_chart.png --caption "雷达图"
   lark-cli docs +media-insert --doc doxcnXXX --file ./factor_chart.png --caption "因子图"
   lark-cli docs +media-insert --doc doxcnXXX --file ./heatmap_chart.png --caption "热力图"
   ```
6. （可选）创建飞书画板：
   - 使用 `lark-whiteboard` skill
   - 生成 whiteboard JSON DSL
   - 使用 `lark-cli docs +whiteboard-update` 上传

**输出**：
- 飞书文档 URL
- 飞书画板 URL（如创建）

**负责**：AI Agent / 用户确认授权

---

4. 更新 SOP（如有改进点）。

**输出**：
- 归档文件夹
- SOP 更新记录

**负责**：用户 / AI Agent

## 四、关键检查点（Checklist）

### 每次分析前
- [ ] 明确分析目标（是判断反转？验证指标？还是出报告？）
- [ ] 至少获取 2 个周期的截图
- [ ] 记录截图中的指标名称和参数

### 指标改造时
- [ ] 确认 MT4 源码编译逻辑
- [ ] MQL5 转换后删除 `IndicatorBuffers()`
- [ ] 使用句柄链路或手动计算替代 `iMAOnArray` / `iRSIOnArray`
- [ ] MetaEditor 5 编译通过

### 量化建模时
- [ ] 雷达图维度不超过 8 个
- [ ] 因子有明确方向（+/-）和置信度
- [ ] 评分公式可解释、可复现
- [ ] 阈值规则清晰

### 等待入场时机（桃子成熟）时
- [ ] H4 SR 支撑已被测试 1~3 次且未有效跌破
- [ ] H1 SR / 枢轴带明显收缩
- [ ] RSIOMA 低位金叉且黄线持续向上
- [ ] 出现收盘站稳 H4/H1 SR 上沿的确认 K 线
- [ ] 止损位明确，风险回报比 ≥ 1:1.5
- [ ] 未来 4 小时内无重大数据/事件
- [ ] 仓位已按止损距离计算好

### 飞书交付时
- [ ] 用户已完成飞书授权
- [ ] 文档已成功创建
- [ ] 图片已插入到正确位置
- [ ] 已添加免责声明

### 复盘时
- [ ] 回测样本量 ≥ 10
- [ ] 失败案例已记录
- [ ] 权重是否需调整已评估

## 六、举一反三：如何迁移到其他品种 / 周期 / 策略

答案是：**可以举一反三**。本 SOP 的核心是“多周期结构 + 动量确认 + 等待成熟入场点”的框架，不绑定某个具体品种或指标。

### 6.1 哪些模块是通用的（可直接复用）

| 模块 | 通用性 | 说明 |
|------|--------|------|
| 多时间框架分析 | ⭐⭐⭐⭐⭐ | 任何流动性市场都适用 |
| SR/枢轴收缩概念 | ⭐⭐⭐⭐⭐ | 支撑阻力是价格行为的共性 |
| “桃子成熟”入场思想 | ⭐⭐⭐⭐⭐ | 等待确认是任何策略的基础 |
| 雷达图评分法 | ⭐⭐⭐⭐ | 维度可根据策略调整 |
| 因子贡献模型 | ⭐⭐⭐⭐ | 因子和权重需根据品种校准 |
| 风险回报比 ≥ 1:1.5 | ⭐⭐⭐⭐⭐ | 通用风险管理原则 |
| 仓位公式 | ⭐⭐⭐⭐⭐ | 任何品种都适用 |

### 6.2 迁移时需要调整的变量

| 变量 | 本次 XAUUSD 设置 | 迁移时如何调整 |
|------|------------------|----------------|
| **品种** | XAUUSD | 可替换为 EURUSD、GBPJPY、原油、股指等 |
| **主周期** | M15 / M30 | 日内交易可用 M5/M15；波段可用 H1/H4；长线可用 D1/W1 |
| **大周期 SR** | H4 SR | 日内用 H1 SR；波段用 D1 SR；长线用 W1/MN SR |
| **小周期 SR** | H1 SR | 日内用 M15 SR；波段用 H1 SR；长线用 H4/D1 SR |
| **动量指标** | RSIOMA(14,21) | 可替换为 RSI(14)、MACD、Stochastic、CCI 等 |
| **入场触发** | H4/H1 SR 上沿 + RSIOMA 金叉 | 可替换为均线金叉、布林带突破、K 线形态等 |
| **止损基准** | H4 SR 下沿 - 1~2 ATR | 用对应大周期的关键支撑/阻力作为基准 |
| **ATR 周期** | 14 | 根据品种波动特性调整，波动大的品种可用 10 或 20 |
| **风险回报比** | ≥ 1:1.5 | 趋势策略可提高到 1:2~1:3； scalp 策略可降低到 1:1 |

### 6.3 不同交易风格下的适配

| 风格 | 周期组合 | 入场触发 | 持仓时间 |
|------|---------|---------|---------|
| **剥头皮 Scalp** | M1 / M5 / M15 | M5 SR 突破 + 动量爆发 | 数分钟 ~ 30 分钟 |
| **日内交易** | M5 / M15 / H1 | H1 SR 突破 + M15 回踩确认 | 30 分钟 ~ 4 小时 |
| **波段交易** | M30 / H1 / H4 / D1 | H4 SR 突破 + H1 回踩确认 | 数小时 ~ 数天 |
| **趋势跟踪** | H4 / D1 / W1 | D1 均线排列 + H4 回调结束 | 数天 ~ 数周 |

### 6.4 常见品种注意事项

| 品种 | 特点 | 注意事项 |
|------|------|---------|
| **XAUUSD** | 波动大、受事件影响大 | ATR 较大，止损要留足空间；避开重大数据 |
| **EURUSD** | 流动性最好、点差小 | 适合小周期；假突破较多，需严格确认 |
| **GBPJPY** | 波动极大 | 仓位要减小；止损距离放大 |
| **USOIL/WTI** | 受库存/地缘影响 | 注意交割日、库存数据时间 |
| **股指（US30/SPX）** | 与美股开盘强相关 | 避开开盘高波动期；注意隔夜风险 |
| **加密货币** | 7×24 交易、波动极大 | 建议只用日线/4H；杠杆要极低 |

### 6.5 指标替换映射表

如果你不想用 RSIOMA，可以按以下映射替换：

| 想观察的维度 | 可选指标 | 入场信号 |
|-------------|---------|---------|
| 动量强弱 | RSI / MACD / Stochastic | 金叉、背离、突破中轴 |
| 趋势方向 | EMA 排列 / ADX / 布林带 | 均线多头排列、ADX > 25、价格沿布林上轨运行 |
| 波动率 | ATR / Keltner Channel / Bollinger Bands | 布林带收缩后扩张、ATR 从低位回升 |
| 支撑阻力 | Pivot Points / Floor Trader Pivots / 关键高低点 | 价格测试支撑反弹、突破阻力 |
| 量能配合 | Volume / VWAP / OBV | 放量突破、价格站稳 VWAP |

### 6.6 迁移后的验证流程

迁移到新品种/周期后，不要直接实盘，先完成：

1. **历史回测**：至少 3~6 个月数据，统计信号胜率、盈亏比、最大回撤
2. **参数优化**：调整 SR 周期、动量指标参数、入场阈值
3. **样本外测试**：用未参与优化的数据验证稳定性
4. **模拟盘验证**：至少 20~30 笔交易，确认执行无误
5. **小资金实盘**：确认滑点、点差、执行速度符合预期

### 6.7 核心不变的原则

无论迁移到哪个品种或周期，以下原则不变：

1. **多周期共振**：大周期定方向，小周期找入场点
2. **结构优先于信号**：先有支撑/阻力/形态，再看指标确认
3. **等待确认**：不要预测，只跟随确认后的价格行为
4. **风险控制第一**：每笔交易先想止损，再想盈利
5. **概率思维**：没有 100% 胜率，只追求长期正期望
